| View | Focal element | Why |

| FormCard (New Assessment) | Run Assessment button (primary fill, full width, 48px) | Sole conversion action; highest contrast element on the card |

## Spacing Scale

Declared values — all multiples of 4px. Executor must use only these values; no ad-hoc pixel values.

| Token | Value | CSS Custom Property | Usage |
|-------|-------|---------------------|-------|
| xs | 4px | `--space-xs: 4px` | Icon-to-label gap, badge padding, chip padding |
| sm | 8px | `--space-sm: 8px` | Compact row padding, input padding (vertical), cell padding (vertical) |
| sm2 | 12px | `--space-sm2: 12px` | PipelineStep row padding (vertical), compact card insets |
| md | 16px | `--space-md: 16px` | Default field margin, card inner padding (horizontal), input padding (horizontal), cell padding (horizontal) |
| lg | 24px | `--space-lg: 24px` | Card padding (all sides), section vertical gap |
| xl | 32px | `--space-xl: 32px` | Page horizontal padding, layout gap |
| 2xl | 48px | `--space-2xl: 48px` | Form centering top margin |
| 3xl | 64px | `--space-3xl: 64px` | Not used in this phase |

Exceptions:
- Touch targets (source toggle buttons, run button): minimum 44px height per WCAG 2.5.5
- Pipeline step left indent for specialist nodes: exactly `--space-xl` (32px) — not `24px` as current code has 24px
- Exception: `--space-sm2: 12px` — introduced for PipelineStep vertical rhythm and compact card insets. Midpoint between 8px and 16px is required because 16px produces excessive row height in the 260px-wide pipeline panel and 8px produces insufficient tap-target separation. Approved exception.

Add to `index.css` `:root {}`:
```css
--space-xs:   4px;
--space-sm:   8px;
--space-sm2:  12px;
--space-md:   16px;
--space-lg:   24px;
--space-xl:   32px;
--space-2xl:  48px;
--space-3xl:  64px;
```

## Typography

Font family for ALL elements: `'Roboto', sans-serif`

Add to `index.css` `body`:
```css
font-family: 'Roboto', sans-serif;
font-size: 14px;
line-height: 1.5;
-webkit-font-smoothing: antialiased;
```

| Role | Size | Weight | Line Height | CSS Class / Usage |
|------|------|--------|-------------|-------------------|
| Display | 28px | 500 | 1.2 | Risk score number, verdict score only |
| Title | 16px | 500 | 1.35 | Header app name, card subject name, finding agent name, VerdictCard section headings, page title "New Assessment" |
| Body | 14px | 400 | 1.5 | Form labels, input values, pipeline step labels, table cells, agent cell text, summary cell text |
| Label | 12px | 500 | 1.4 | Section sub-labels (ALL CAPS + letter-spacing), badge text, elapsed time, table column headers, run ID line (at opacity 0.45), helper text under Drive URL field |

Notes on collapsed roles:
- Former Heading (22px) is merged into Title (16px). The "New Assessment" page title and VerdictCard section headings now use Title (16px / weight 500).
- Former Caption (11px) is merged into Label (12px). Decorative / secondary label text (run ID, helper text) uses Label role at `opacity: 0.45` to reduce visual weight.
- All former 13px usages (agent cell, progress text, button text, error message monospace) are mapped to Body (14px / weight 400).

Rules:
- Maximum 4 sizes active in any single view (Display, Title, Body, Label)
- Two permitted weights: 400 (body text), 500 (titles, labels, interactive elements, badges, chips, table headers)
- Heading/Display sizes use weight 500, not 700, for Google's visual register
- ALL CAPS text restricted to: label role, severity badge, verdict badge, table column headers
- Letter spacing: ALL CAPS labels `0.08em`; ALL CAPS table headers `0.06em`; normal text `0`
